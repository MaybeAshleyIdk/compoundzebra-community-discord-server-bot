| no markdown-formatting characters |        no          |          no            |          yes

contentDisplay is really just the worst of all worlds smh

allows us a custom pattern matching languages thats more based on words


argument types:
	String  - generic string whose whitespace gets trimmed and squeezed
	Integer - a 64-bit signed integer (parsed as arbitrary long, if too big it's invalid)
	User    - a reference to a user. may either be the snowflake id or the mention format ("<@id>")
	Channel - a reference to a channel. may either be the snowflake id or the mention format ("<#id>")
	Message - a reference to a message in a channel. must be the permalink to the message ("https://discord.com/channels/guild_id/channel_id/message_id")
	Raw     - generic string that is given raw without any modifications (can only be last, leading whitespace is still removed)
